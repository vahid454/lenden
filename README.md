# LenDen — Production Ready (All 4 Phases)

> **Apna hisaab, apni marzi** — Personal & Business Credit Tracker for India

---

## ✅ What's Built

### Phase 1 — Foundation & Auth
- Splash with animated logo
- Phone input (+91 format) → Firebase OTP (6-digit Pinput)
- Auto resend with 30s cooldown
- New user → Profile setup (name + business name → Firestore)
- GoRouter with auth guard
- Material 3 theme — light & dark mode

### Phase 2 — Customer Management
- Real-time customer list (Firestore stream)
- Add / Edit / Delete customers (cascade deletes transactions)
- Balance chips: green = owe you, red = you owe them
- Debounced prefix search, shimmer loading, empty states
- Customer detail with quick actions (Call, SMS, Copy)

### Phase 3 — Transaction Ledger
- Add / Edit / Delete transactions (atomic Firestore balance update)
- Type toggle: You Gave / You Got
- Running balance — always consistent via `runTransaction()`
- Month-grouped ledger with monthly subtotals
- Date picker, note field
- Full customer detail page with collapsible header

### Phase 4 — Reports, Export, Offline, Polish
- Reports page with period filters (Today, Week, Month, Custom)
- Bar chart (monthly overview) + Line chart (daily activity)
- PDF export — professional A4 ledger with customer & transaction data
- WhatsApp sharing — pre-formatted ledger message to customer's number
- General share sheet (share_plus)
- Offline indicator banner (connectivity_plus)
- Hive local cache for offline reads
- Biometric lock (local_auth)
- Clear cache option
- NavigationBar (Material 3) with 3 tabs
- Sorted customer list (by highest balance)
- Profile tab with stats, settings, version info

## 🔥 Firebase Setup

### 1. Create Project
1. https://console.firebase.google.com → **Add Project** → name `lenden-app`
2. Disable Google Analytics (optional)

### 2. Enable Phone Auth
Firebase Console → **Authentication** → **Sign-in method** → Enable **Phone**

### 3. Create Firestore
Firebase Console → **Firestore Database** → **Create database** → **Start in test mode** → Region: `asia-south1`

### 4. Add Android App
1. Firebase Console → **Project Settings** → **Add app** → Android
2. Package name: `com.yourcompany.lenden`
3. Download `google-services.json` → place at `android/app/google-services.json`

### 5. FlutterFire CLI
```bash
dart pub global activate flutterfire_cli
flutterfire configure --project=lenden-app
# This auto-generates lib/firebase_options.dart
```

### 6. Deploy Security Rules & Indexes
```bash
npm install -g firebase-tools
firebase login
firebase init firestore
# Copy rules from firestore/firestore.rules
# Copy indexes from firestore/firestore.indexes.json
firebase deploy --only firestore:rules,firestore:indexes
```

### 7. SHA-1 for Phone Auth (Required!)
```bash
cd android && ./gradlew signingReport
# Copy SHA-1 from "Variant: debug"
# Firebase Console → Project Settings → Your Android App → Add fingerprint
```

  balance (atomic ± on every transaction),

## 🏪 Play Store Deployment

### Step 1 — Keystore
```bash
keytool -genkey -v -keystore ~/lenden-release.keystore \
  -alias lenden -keyalg RSA -keysize 2048 -validity 10000
```

### Step 2 — key.properties
Create `android/key.properties`:
```properties
storePassword=YOUR_STORE_PASSWORD
keyPassword=YOUR_KEY_PASSWORD
keyAlias=lenden
storeFile=/path/to/lenden-release.keystore
```

### Step 3 — Reference in build.gradle
```gradle
def keystoreProperties = new Properties()
keystoreProperties.load(new FileInputStream(rootProject.file('key.properties')))

android {
    signingConfigs {
        release {
            storeFile     file(keystoreProperties['storeFile'])
            storePassword keystoreProperties['storePassword']
            keyAlias      keystoreProperties['keyAlias']
            keyPassword   keystoreProperties['keyPassword']
        }
    }
    buildTypes {
        release { signingConfig signingConfigs.release }
    }
}
```

### Step 4 — Build App Bundle
```bash
flutter build appbundle --release
# Output: build/app/outputs/bundle/release/app-release.aab
```

### Step 5 — Upload
1. https://play.google.com/console → Create app
2. Upload `.aab` to **Internal testing** first
3. Fill store listing (screenshots, description)
4. Add SHA-1 of release keystore to Firebase
5. Promote to Production

## 🔒 Firebase Production Tips

```javascript
// Firestore Rules — already in firestore/firestore.rules
// Key rules:
// • Users can only read/write their own documents
// • Amount validated server-side: 0 < amount <= 10,000,000
// • Transaction type must be 'gave' or 'got'
```

### Enable App Check (Recommended for Production)
```bash
# Firebase Console → App Check → Register your app
# Add to main():
await FirebaseAppCheck.instance.activate(
  androidProvider: AndroidProvider.playIntegrity,
);
```

### Enable Firestore Offline Persistence
```dart
// Already handled by Firestore's default settings on Android
// Manually set cache size:
FirebaseFirestore.instance.settings = const Settings(
  persistenceEnabled: true,
  cacheSizeBytes:     Settings.CACHE_SIZE_UNLIMITED,
);
```

## 🐛 Common Issues

| Problem | Fix |
|---|---|
| OTP not received | Add SHA-1 debug fingerprint to Firebase Console |
| `google-services.json not found` | Download from Firebase → Project Settings → Your Android App |
| `MissingPluginException` | `flutter clean && flutter pub get` |
| Firestore `permission-denied` | `firebase deploy --only firestore:rules` |
| `failed-precondition` (index missing) | `firebase deploy --only firestore:indexes` |
| PDF export fails | Check `WRITE_EXTERNAL_STORAGE` permission granted |
| WhatsApp not opening | Ensure `<queries>` block in AndroidManifest.xml |
| Biometric not working | Add `USE_BIOMETRIC` permission to manifest |
| `minSdkVersion` error | Set `minSdkVersion 21` in `android/app/build.gradle` |
